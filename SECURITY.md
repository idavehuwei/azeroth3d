# 安全策略

我们认真对待 Azeroth3D 项目的安全性。

## 报告漏洞

**请勿通过公开的 GitHub Issue 或讨论区报告安全漏洞。** 公开披露可能让所有用户面临风险。

请通过以下方式私下报告：

- **GitHub 安全公告（推荐）。** 前往 [Security tab](https://github.com/idavehuwei/azeroth3d/security/advisories/new) 创建草稿公告。这将使报告在修复之前保持私密。

请尽可能提供：

- 问题描述和影响范围
- 复现步骤
- 受影响的版本或提交

## 已知的安全注意事项

由于 Azeroth3D 是一个纯前端单页游戏，以下安全要点需要注意：

1. **存档导入/导出。** 存档数据为 JSON → Base64 格式，导入时验证数据结构完整性。
2. **localStorage 存储。** 游戏进度存储在浏览器本地，不会上传到任何服务器。
3. **无网络请求。** 当前版本（v1.4）不涉及任何网络通信，不存在联机安全风险。
4. **Three.js CDN。** 游戏从 CDN 加载 Three.js，请确保版本固定（r128）以防止供应链攻击。

## 未来联机版本

当项目发展到 v3.0 联机阶段时，将制定更完善的安全策略，涵盖：

- 服务器权威性（客户端只发意图，服务器决定结果）
- 输入验证
- 反作弊措施
- 数据加密

## 预期响应时间

我们会在 48 小时内确认收到报告，并尽快发布修复。
