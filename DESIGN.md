# 熔火之心 · 设计语言

**状态：** 采用中。设计变更按阶段推进，非孤立片段。
**范围：** 游戏客户端（`game.html` 入口）。
**更新：** 2026-07-21

本文档是 Azeroth3D 游戏界面视觉风格和交互体验的权威来源。它规定了色彩、排版、UI 组件和交互反馈的标准，确保所有界面元素一致、协调。

---

## 1. 设计原则

1. **世界优先。** 3D 场景是核心。永久 UI 贴边，使用受控的半透明，不覆盖中央游戏区域。
2. **经典结构，简洁表达。** 血条、技能栏、日志、任务追踪放在玩家期望的位置，但使用更简洁的现代样式。
3. **锻造的奇幻感，而非玻璃 UI。** 避免通用的扁平卡片、霓虹渐变、过度模糊。表面应呈现黑曜石、岩浆和青铜质感。
4. **金色是结构性的。** 金色用于边缘、选中态、奖励和层级标识，不填充大面积。
5. **清晰胜于装饰。** 当更多装饰和更多可读性冲突时，可读性胜出。

## 3. 排版

- 字体：`"Noto Sans SC", "Microsoft YaHei", sans-serif`（系统字体）
- 尺寸：UI 文字 9-20px，公告 22-40px，伤害飘字 14-18px
- Banner 标题：`font-weight: 900`，`letter-spacing: 6-10px`
- 阴影：`text-shadow` 用于增强可读性，Boss 名称和公告使用橙色/红色辉光

## 4. UI 组件

### 血条系统

- 设计：顶部 BOSS 血条居中，玩家血条左下角
- 结构：`.barShell`（外框）+ `.barFill`（填充）+ `.barText`（文字）
- 样式：边框 1px `--ui-line`，背景 `rgba(0,0,0,.65)`，内阴影
- 动画：`transition: opacity .8s`（Boss 血条出现/消失）

### 技能栏

- 位置：底部居中
- 尺寸：58×58px，间距 10px
- 状态：冷却遮罩（`.oncd`）、GCD 遮罩（`.gcd`）
- 交互：`active` 态缩放 `.93`

### 对话/拾取按钮

- 位置：血条下方居中
- 样式：金色渐变按钮，与 `--gold` 色系一致
- 切换：自动在「拾取」和「对话」间切换

### 对话框

- 位置：底部居中
- 样式：`--ui-bg` 半透明背景，`--gold` 边框
- 按钮：全宽按钮，悬停高亮

### 战斗日志

- 位置：右下角
- 最大条目：9 条
- 效果：`-webkit-mask-image` 渐变渐变遮罩（底部淡出）

### 弹窗（开始/结束）

- 全屏遮罩，径向渐变从 `rgba(60,15,0,.88)` 到 `rgba(5,2,1,.96)`
- 过渡：`opacity .6s`

## 5. 动画与反馈

### 公告（announce）

- 位置：屏幕上方 24%
- 动画：`pop` 关键帧（缩放 0.6→1.05→1 + 透明度 0→1→0）
- 持续时间：3 秒

### 伤害飘字（FCT）

- 位置：场景坐标投影到屏幕
- 动画：`fct` 关键帧（缩放 0.7→1.15→0.95，上移 0→70px）
- 持续时间：1.1 秒

### 受击闪红

- 屏幕边缘径向渐变红色
- 过渡：`opacity .9 → 0` 持续 0.5 秒

### 粒子爆发

- 位置：事件发生点
- 材质：`AdditiveBlending`，半透明
- 衰减：1.1 秒后自动移除

### 生物动画语言（plan-v1 · V1-A3）

- **原则：** 纯表现，不改伤害 / 仇恨 / 碰撞；幅度与频率只进 `BALANCE.anim`。
- **状态：** `idle` / `walk` / `attack` / `cast` / `dead`（见 `userData.anim.state`）。
- **幅度上限：** 腿摆 `walkAmp ≤ 0.75`；转弯外倾 `|z| ≤ 0.2`；禁止整模 360° 空翻。
- **死亡：** 必须 `beginDeathRoll` / `beginFinalSink` 插值，禁止瞬切 `rotation.z`；可有落地顿挫（`thumpBoost`）。
- **挂点：** 四足 `legs[]` + `head`；人形 `leg/arm`；龙 `wingL/R` + `tail`；加新怪先挂点再摆。
- **调试：** `?anim=1` 左上角叠状态字；`BAL.anim.enabled=false` 关闭驱动（死亡插值仍可收尾）。

### 音频设计语言（plan-v1 · V1-A5）

- **原则：** 零 `.mp3/.ogg/.wav`；调音只改 `sfx.js` 的 `SOUNDS` / `MUSIC` / `AMBIENCE` 与 `BALANCE.sfx`。
- **分轨：** `sfx` / `music` / `ambience` 独立 Gain；总静音仍走 mute 按钮。
- **材质脚步：** 区默认表面（草/土/石/渣）+ 营地木板兴趣点；雨天草/土走 `grass_wet`；`SFX.playFoot`。
- **受击分层：** `playHit(type)` → 肉体/甲壳/龙鳞/元素/玩家护甲；叠音 ≤3。
- **调音台：** 独立打开 `sfx_studio.html`（列表试听、改 JSON、导出、分轨滑条）。

- 技能按钮点击（非悬停）交互

## 7. 工程约束

- **零外部资源。** 所有图标由 Canvas 2D 运行时绘制，不提交 .png 文件。
- **品质色由参数驱动。** 图标描边、物品名颜色、掉落方块光效都由 `QUALITY` 表定义。
- **性能。** 粒子爆发、掉落方块等动态对象在生命周期结束后自动 `dispose`。
- **CSS 变量。** 主题色通过 CSS 自定义属性定义，便于统一调整。
