## 二 · v4 的三条纪律（决定抄什么、不抄什么）

**① 保真抄公式，体量不抄。**
WoC 之所以「一打就是那个味道」，不是因为它有 60 个任务，而是因为怒气怎么涨、多少几率被躲闪、护甲吃掉多少伤害都是初版公式。公式层是纯函数、几十行、可单测、零美术成本——**这是整个 v4 里投入产出比最高的一步**。而九职业、三区域、五副本是纯内容体力活，我们用两个区域、六职业、两个副本就能撑住同样的结构。

**② 联机的门票是纯 sim 层，不是服务器。**
WoC 的「一套核心跑三个宿主」不是架构洁癖，是它能同时有离线、联机、机器人的唯一原因。我们 STEP 0/1 已经在朝这个方向走，v4 要把它走完：`js/sim/` 目录下的文件**不许出现 `THREE`、`document`、`window`、`Math.random`、`Date.now`**，并用一个 grep 脚本当守卫。做完这一步，联机、AI 队友、自动化机器人三件事都会同时变便宜；不做这一步，它们三个都做不了。

**③ 能被机器人跑通的，才算做完。**
WoC 的五个机器人能自己打穿 Hollow Crypt，那才叫「这个副本成立」。我们把 `test_step01.js` 的 vm 沙箱扩成套件，再加一个浏览器代理接口 `window.__game.controller`，每个 STEP 的验收都写成一条命令。**验收标准里没有命令的 STEP，不许合并。**

### STEP 15 · 经典公式层 `P0 · v4 的主菜` · ✅ 已完成（2026-07-22）

- **前置决策：等级上限**。建议定 **1–12**（配两个区域：1-7 / 6-12），不追 WoC 的 20 级——所有公式都吃等级，先定死再写。
- 在 `sim/rules.js` 落地五组纯函数，全部可单测：
  1. **命中表**：按等级差的未命中 / 躲闪 / 招架；法术命中带 +3 级断崖（96 / 95 / 94 / 83%）。
  2. **护甲减伤**：`armor / (armor + 85·攻击者等级 + 400)`。
  3. **怒气换算**：`c = 0.0091L² + 3.23L + 4.27`；造成伤害得 `7.5·d/c`，受到伤害得 `2.5·d/c`。战士的怒气从此不再是「打一下 +8」。
  4. **属性转换**：耐力前 20 点每点 1 生命、其后每点 10；智力前 20 点每点 1 法力、其后每点 15。装备属性从此有意义。
  5. **经验曲线**：升级所需 400 / 900 / 1400…；怪物经验 `45 + 5·等级`，并保留**灰怪零经验**波段（逼玩家往新区域走，这是「世界感」的来源之一）。
- 战斗节奏三件套：**1.5 秒公共冷却**、**武器挥击计时器**（自动攻击不再是固定 `autoSpd`）、**5 秒回蓝规则**（施法后 5 秒内不回法力）。
- 唯一插入点仍是 `hitEntity`：命中判定 → 护甲减免 → 暴击 → 吸收盾 → 扣血 → 回调。战斗日志相应细化（未命中 / 躲闪 / 暴击）。

> **WoC 对照**：README 的 “Classic-fidelity 检查清单” 就是这一步的需求文档，逐条照抄即可。

> **验收**：`node test_rules.js` 覆盖五组公式的边界值（等级差 ±3、护甲 0 与极大值、怒气首次击打、灰怪经验为 0）；实机 5 级打 1 级野猪不掉经验、打同级怪偶尔 miss；战士打满怒气的时间与改前处于同一量级（不能因为公式变了就打不出技能）。

## 七 · 阶段 E · v4.0 联机与社交（STEP 25–28）

> 这一段是可选的分水岭。做之前先回答一个问题：**你想要的是「和朋友一起打」还是「一个能自托管的世界」？** 前者到 STEP 27 就够了，后者才需要 STEP 26 的账号体系。

### STEP 25 · 权威服务器最小可用版 `P0（本阶段闸门）`

- `server/main.js`：Node + `ws`，**20 Hz** tick。服务器用 `vm` 沙箱加载 `js/sim/**`（手法与 `test_step01.js` 完全一致，因此**不需要打包器、不需要改成 ESM**）。
- 客户端只发**意图**（移动标志 + 朝向 + 技能编号），服务器跑世界并下发按兴趣范围裁剪的快照；所有战斗结算、掉落掷骰、任务计数只在服务器发生。
- 客户端抽出 `IWorld` 风格的读取接口：离线时读本地 `Sim`，联机时读 `ClientWorld`（快照镜像），渲染层对两者无感。
- 先只做**局域网双开**：两个标签页能互相看见、看见对方打怪、看见对方的名字。
- 持久化先用**单个 JSON 文件**（每 30 秒 + 退出时落盘），不要一上来就上数据库。

> **WoC 对照**：客户端以 20 Hz 流式发送移动意图 + 命令，服务器运行一个共享 `Sim` 并发送约 120 码兴趣范围内的快照；客户端只是一个渲染器。

> **验收**：`npm run server` + 两个浏览器窗口，A 打死的野猪在 B 屏幕上也倒下；断线重连后角色状态还在；关掉服务器再开，进度不丢。

### STEP 26 · 账号 · 角色 · 自托管 `P2`

- 账号：`scrypt` 哈希密码 + 有效期令牌；认证端点按 IP 限流。每账号最多若干角色，名字全局唯一。
- 持久化从 JSON 升级为 **SQLite**（`better-sqlite3`，零运维）——除非你真的要多人上量，否则不必上 Postgres。
- `docker-compose.yml` 一条命令起服务；`ALLOW_DEV_COMMANDS` 环境变量控制作弊指令（**生产环境必须关**）。
- 启动页加「在线游玩 / 离线游玩」双入口，离线路径必须保持可用（双击 HTML 就能玩，这是我们相对 WoC 的优势，别丢）。

> **WoC 对照**：scrypt 密码 + 7 天令牌、角色以 JSONB 存 Postgres（每 30 秒 / 登出 / 关服各存一次）、一条 docker compose 命令自托管、TLS 反代两行 Caddy 配置。

> **验收**：新机器上 `docker compose up -d` 后能注册、建号、进世界；`ALLOW_DEV_COMMANDS=0` 时作弊指令全部失效。

### STEP 27 · 小队与拾取权 `P1`

- 小队（≤5）：右键玩家邀请，左侧小队框架（复用 STEP 20 组件），`/p` 频道。
- **经典拾取权**：第一个对怪物造成伤害的玩家拥有掉落 / 经验 / 任务进度；他人拾取被拒。受害者死亡后怪物重新锁定下一个攻击者。
- 组队经验按初版系数分摊：3 / 4 / 5 人对应 1.166 / 1.3 / 1.43。
- 小队共享击杀任务进度；治疗可点小队框架指定目标（STEP 16 的光环系统在这里第二次收租）。

> **WoC 对照**：小队最多 5 人、共享拾取权与击杀任务进度、按真实初版加成分摊经验、`/p` 小队聊天、小地图蓝点。

> **验收**：`node scripts/party_e2e.js` 两个客户端组队打同一只怪，经验按系数分摊、非拥有者拾取被拒。

### STEP 28 · 决斗 → 交易 → 竞技场 `P2 / P3`

- **决斗**（先做，最便宜）：右键挑战，3 秒倒计时，血量降到 1 点结束，无人死亡，跑开 60 码判负，区域公告胜者。
