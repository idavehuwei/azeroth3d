# 熔火之心 · plan-V3 —— 经典化对齐计划

**CLASSIC-FIDELITY PLAN — 对齐初版魔兽世界的「玩法手感」，对齐 WoC 的「工程架构」**

> 基线：代码实地检查结果（见第〇节），`core / models / world / combat / main` 五模块 + `test_step01.js`，STEP 0（BALANCE + SeededRng）与 STEP 1（`hitEntity` 统一受击）已落地
> 双参考：**暴雪初版**提供公式与系统清单；**levy-street/world-of-claudecraft** 提供「怎么在一个 Three.js 单页项目里把这些落地」的工程答案

## 〇 · 代码检查结论

### 0.1 逐文件盘点

| 文件 | 现状 | 与「MMO 骨架」的距离 |
| --- | --- | --- |
| `core.js` | `$ / clamp / rand / R`、`BALANCE` 六大类数值、`SeededRng + worldRng + srand`、渲染器、相机、`sceneRaid` 熔岩环境（shader 岩浆 + 火星粒子）、`makeLabel` | 地基**合格**。`BALANCE` 已经是 content-as-data 的雏形，但只装了「这一场 Boss 战的数字」，没有属性、等级、命中表的位置 |
| `models.js` | `buildPlayer / buildMage / buildArcher / buildBoss / buildElder / buildBoar / buildFlameSpawn` 七个硬编码函数 | 无骨架层级、无动画状态机、`userData` 只有 `{armR,armL,legR,legL,cape}` 五个平铺挂点。**plan-V2 的 R5/R6 已针对此出方案，本计划不重复** |
| `world.js` | 莫高雷单区域：草地圆盘 + 台地 + 树石 + 营地 + 传送门；`MOBS` 五只野猪（wander/aggro/leash/respawn 状态机）；长老 NPC；`QUEST = {state, kills}` | 野怪 AI 已有**四分之一**的初版清单；任务是一条写死的三段状态机，不是任务系统 |
| `combat.js` | `S` 全局状态；`CLASSES` 三职业（战士/法师/弓箭手）各 4 技能 + 自动攻击；`hitEntity` 统一受击；`bossAI` 四技能两阶段 + 潜入召唤小怪；投射物、地面 AoE 预警、粒子爆发 | Boss 战本身**做得相当扎实**（读条、预警圈、阶段切换都有）。但战斗数学只有一行 `rand(0.92,1.08)` |

### 0.2 一句话诊断

> **现在的东西是「一个草原任务 + 一场做得不错的 Boss 战」，不是一个 MMO 的骨架。**

画面问题 plan-V2 已经有答案。plan-V3 要解决的是：一个初版 WoW 玩家打开它，**手上的感觉**从第一秒就不对。

## 一 · 对齐原则：抄什么、不抄什么

### 1.1 抄公式与系统（这是 WoC 的核心示范）

WoC 的 README 有一节叫「Classic-fidelity 检查清单」，逐条列出它实现的初版公式。**这是一份现成的施工图**，我们照着做即可，不必自己考据。它明确列出的包括：怒气换算公式、带 +3 级断崖的法术命中表、护甲减伤公式、耐力/智力的分段换算规则、经验曲线与怪物经验、1.5 秒公共冷却与武器挥击计时器、五秒法力规则。

### 1.2 抄架构（`sim` 与渲染分离）

WoC 的 `src/sim/` 是**零 DOM、零 Three.js 依赖**的确定性核心，同一份代码跑离线浏览器、联机服务器和测试机器人。我们的体量不需要完全分层，但**公式必须先落到纯函数层**——因为公式是最需要单测的部分，而它恰好也是最容易做成纯函数的部分。这一步的红利立刻兑现：`test_step01.js` 的基座直接能跑几百条公式断言。

### 1.3 不抄名字与美术 —— 建议做一次原创化

WoC README 结尾写得很清楚：**名字、任务和各个区域都是原创的；公式与机制则遵循初版**。这不是客气话，是它能公开发布、被 800+ 人 star、被 fork 242 次的前提。

我们目前的专有名词全部是暴雪的注册商标或作品元素：莫高雷、熔火之心、拉戈斯（音译 Ragnaros）、萨弗拉斯、烈焰之子、牛头人图腾柱。**建议在 C0 做一次批量替换**——玩法一行不改，只换名字与文本。这既让项目能安心公开，也逼着我们把文本抽成数据表（`STRINGS` / `NAMES`），是重构的顺手收益。

> 参考原创化方向：草原区域 → 「赤蹄草甸」；副本 → 「炽心熔窟」；Boss → 「熔渊之王 · 卡尔戈」；橙武 → 「熔渊之柄」；小怪 → 「火裔」。风格保留，指向替换。

## 二 · 五条轨道与总体架构

plan-V3 的步骤按**轨道**组织，不是严格串行 —— 每条轨道内部有依赖，轨道之间大多可并行或交替。

```
Track A · 手感   C1 操作与相机 ── C2 目标系统 ─────────────────┐
Track B · 数学   C3 sim 分层 + 属性 ── C4 命中表 ── C5 资源    ├─→ C11 副本 ── C12 AI 队友
Track C · 成长   C6 等级经验 ── C7 法术等级 ── C8 装备纸娃娃    │
Track D · 世界   C9 任务系统 ── C10 死亡/生活 ── C13 第二区域   │
Track E · UI     贯穿全程，每步交付对应的经典界面组件 ──────────┘
```

**目标文件结构**（在 plan-V2 的渲染分层之外，新增一个 `sim` 层）：

```
js/
├─ sim/                       ← 新增：零 DOM / 零 THREE，可在 Node 直接跑
│  ├─ formulas.js   命中表 · 护甲减伤 · 暴击 · 怒气 · 法力回复 · 经验曲线
│  ├─ stats.js      五属性 → 派生属性（HP / MANA / AP / 暴击 / 闪避）
│  ├─ content.js    CLASSES · SPELLS（含等级）· MOBS · ITEMS · QUESTS · TALENTS
│  └─ entity.js     统一实体：属性块 + 资源 + Buff 列表 + hitEntity 结算
├─ ui/                        ← 新增：经典 HUD 组件（纯 DOM + CSS）
│  ├─ frames.js     玩家 / 目标 / 目标的目标 / 小队框架
│  ├─ windows.js    角色面板 · 法术书 · 背包 · 任务日志 · 商人 · 拾取
│  ├─ tooltip.js    金边工具提示（物品 / 法术 / Buff 通用）
│  └─ nameplate.js  头顶姓名板 + 血条 + 精英边框
├─ (plan-V2 的渲染层：palette / textures / terrain / props / sky / rig / creatures / vfx)
├─ core.js · world.js · combat.js · main.js · debug.js
└─ tests/  test_step01.js（已有）· test_formulas.js · test_progression.js（新增）
```

**铁律（三条，与前两版一致）**：零素材文件；渲染层不碰数值；摆放随机走 `srand`、玩法随机走 `rand`。
**新增第四条**：`js/sim/` 内禁止出现 `document` / `THREE` / `Math.random` / `Date.now`。用 `npm test` 在 Node 里跑得通，就是这一层合格的唯一标准。

### C1 · 经典操作与相机 `P0` · 1-2 次迭代 · ✅ 已完成（2026-07-22） · **本计划投入产出比最高的一步**

**现状（完成前）**：球坐标雏形与 WASD/QE/左右键视角已有；缺跳跃、自动跑、相机碰撞、第一人称、移动端右半屏/捏合、输入意图层。

**目标：初版键位全套**

| 输入 | 行为 |
| --- | --- |
| `W` / `S` | 前进 / 后退（沿角色朝向） |
| `A` / `D` | 转向（**按住鼠标右键时改为横移**——这个细节是初版手感的关键） |
| `Q` / `E` | 横移 |
| 鼠标右键拖动 | 鼠标视角：转动镜头**同时转动角色朝向** |
| 鼠标左键拖动 | 环绕镜头：只转镜头，角色朝向不动 |
| 滚轮 | 缩放（3m 到 25m，最近处自动切第一人称视角） |
| `Space` | 跳跃（抛物线，落地判定走 `heightAt`） |
| `R` | 自动奔跑开关 |
| `Esc` | 取消目标 / 关闭最上层窗口 |

**落地**：`BALANCE.camera` / `BALANCE.move`；`getMoveIntent()`；`resolveCamCollision`；触控右半屏 + 捏合。

**实现要点**

- 相机改为**球坐标跟随**：`{ yaw, pitch, dist }` 三个状态量，目标点 = 角色胸口高度。`pitch` 钳制在 `-1.4 ~ 0.6` 弧度，`dist` 钳制在 `3 ~ 25`。
- 相机碰撞：从目标点向相机位置射一条 `Raycaster`，命中地形或岩石时把 `dist` 临时压近（初版就是这么做的，没有这个会频繁穿墙）。
- 移动改为**朝向相对**：`forward = (sin(face), 0, cos(face))`，`face` 由 A/D 或鼠标视角驱动，不再是「按 W 就往屏幕上方走」。
- 跳跃：`vy` 初速 + 重力积分，落地取 `heightAt(x,z)`（若 plan-V2 的 R2 尚未做，则地面高度恒为 0，接口先留着）。
- **移动端不砍**：保留摇杆，右半屏拖动 = 鼠标视角，双指捏合 = 缩放。两套输入映射到同一组意图（`{forward, back, strafeL, strafeR, turnL, turnR, jump}`），这也是 WoC 浏览器代理驱动 `controller.move()` 的做法 —— 顺手让自动化测试可以驱动角色。

**验收**

- 按住右键转一圈，角色跟着转，松开后 W 沿新朝向走。
- 滚轮从贴脸拉到最远无穿模、无抖动；镜头贴墙时自动拉近而不是钻进石头里。
- 跳跃落地不穿地、不弹跳。
- 手机上单手能走能转。

**风险**：低。纯输入与相机层，不碰任何战斗逻辑。但改完**必须**跑一遍完整回归，因为技能的朝向判定（`heroicStrike` 的 `reach`、`charge` 的贴近）依赖 `S.p.face`。

### C2 · 目标系统 + 姓名板 + 目标框 `P0` · 1 次迭代 ✅

**落地**：`S.currentTarget` / `S.target`；`resolveSkillTarget` / `cycleHostileTargets`；`#targetFrame` + `js/ui/frames.js` / `js/ui/target.js`；`V` 姓名板全显 + 仇恨着色；右键短按攻击。

**改动**

- `S.target` 全局当前目标（与 `S.currentTarget` 同步）。技能优先当前目标，否则自动选取，再否则「你没有目标。」
- **Tab 循环**：视锥内 + 距离排序；`Shift+Tab` 反向。
- **点击选取**：`Raycaster` 打到怪物模型或姓名板即选中。
- **目标框（`js/ui/frames.js`）**：名字、等级、血条、精英/稀有边框、目标的目标小框。`#bossFrame` 仍作 Boss 战顶部大条。
- **姓名板**：沿用 `core.js` `makeNameplate` / `updateNameplatePresentation`；`V` 全显；距离衰减；仇恨中名字变红（遮挡 raycast 不做，同 R7 性能取舍）。
- **右键行为**：短按敌人 = 选中并攻击节奏；否则 `tryInteract`（NPC/拾取）。

**验收**：Tab 能在五只野猪之间循环；选中后目标框显示名字、等级、血量；打谁掉谁的血；Esc 取消目标后技能提示「你没有目标」。

## 五 · Track C · 成长

### C6 · 等级与经验（真曲线） `P0` · 1 次迭代 ✅

**落地**：`js/sim/xp.js` + `SIM_CONTENT.xp`（`XP_CURVE` 400/900/1400…→20 级）；野怪 `45+5×Lv` + 等级差倍率 + 灰色线（6 级打 1 级猪零经验）；休息经验池（离线/营火/旅店）+ 经验条蓝段；升级满血满资源 + 全屏微光。

**验收**：从 1 级打到 5 级节奏合理；打 1 级野猪到 6 级时经验归零；升级瞬间有强反馈。

### C7 · 法术书与技能等级 `P1` · 1 次迭代 ✅

**落地**：各技能 `unlock` 等级；1 级仅学会首技能并自动上动作条；升级弹出「学会了新法术」；`S.actionBar` 拖拽绑定（法术书 → 槽）；Rank 仍走 `getSkillBal`；法术书已学会/全部分页 + 悬停 tip；动作条射程红 / 资源蓝 / CD 扫描。

**验收**：1 级角色只有 1 个技能；升到解锁等级弹出「学会了新法术」提示；法术书里拖技能到动作条生效。

### C8 · 物品 · 装备 · 纸娃娃 `P0` · 2 次迭代 ✅

**落地**：品质含灰（`poor`）；装备槽扩至腰/腕/副手/远程；`applyEquipStats` → `S.p.equipStats` → `mergeStats` → `deriveStats`（AP/暴击进 C4）；绿武带 `str/agi`/`dmgRange`/`ilvl`；`js/ui/tooltip.js` 金边 tip；背包右键装备 + 品质边框；C 纸娃娃显示五维/AP/暴击并可拖动旋转预览；主手 `setWeapon` + 普攻走武器伤害区间。

**验收**：拾取绿色武器 → 背包可见 → 装备 → 角色面板攻击强度上升 → 打怪伤害变高（C4）→ 手上模型换成新武器 → 卸下还原。

## 六 · Track D · 世界

### C9 · 任务系统（数据化） `P0` · 1-2 次迭代 ✅

**落地**：以现有 `QUESTS` 为 `QUEST_DB`（不搬迁大表）；目标别名 `collect→deliver` / `explore→arrive` + 新 `interact` 地面闪光物；`activeMax:5`；灰色感叹号（等级不够）；L 日志分「进行中/已完成」+ 地图标记；`rewards.choice` 职业优先自选；样例任务 `camp_cache` 证明「只写数据」。

**验收**：加任务只写 `QUESTS` 条目；日志可同时挂 5 条；放弃后可再接；地面箱可 F/右键调查；交任务可选奖励。

### C10 · 死亡 · 墓地 · 尸体跑 · 进食饮水 `P1` · 1 次迭代

- **死亡流程**：倒地（接 plan-V2 R6 的 `die` 动画）→ 弹出「释放灵魂」→ 灵魂形态（半透明、灰白世界、移速 +50%）在最近墓地重生 → 跑回尸体 → 「复活」（半血半蓝）。
- 可选：远程复活（在墓地精灵处直接复活，代价是**虚弱状态**，一段时间内属性大幅下降）—— 初版的这个惩罚设计是「小心一点」的动机来源。
- **坐下进食 / 饮水**：坐下后 18 秒内逐步回复生命 / 法力，受到伤害或站起来中断；食物与水从商人购买，是背包里的常驻消耗品。
- 摔落伤害：下落高度超过阈值按比例扣血（依赖 plan-V2 R2 的 `heightAt`）。

**验收**：死一次能完整走完释放 → 跑尸 → 复活；进食途中被野猪打断会提示；从台地跳下来会摔血。

### C11 · 野怪 AI 完整清单 + 精英 + 稀有 `P1` · 1 次迭代

现有 `MOBS` 状态机已有 wander / 固定半径 aggro / leash / respawn 四项。补齐初版清单：

- **按等级差的接近仇恨**：比你高的怪从更远处发现你，比你低很多的怪根本不主动打你（灰怪）。
- **社交拉怪**：同群 ID 的怪一只被打全群跟进；不同族群拉怪半径不同（某些族群拉得特别远——初版著名的「带上朋友」体验）。
- **栓绳 → 脱战 → 满血重置**：现有 leash 规范化，加脱战回血速度、重置期间免疫伤害。
- **精英缩放**：约 2.3× 生命、1.5× 伤害、双倍经验，姓名板金龙边框（数值直接抄 WoC 已调过的比例）。
- **稀有刷新怪**：长计时器、金色名字、必掉优秀物品。
- **中立怪**：不主动攻击，被打才还手（陆行鸟一类）。

**验收**：1 级角色走过 5 级狼群会被追，10 级角色走过同一群不会；打一只狼全群上；稀有怪按小时级计时刷新。

### C13 · 第二个区域 + 小地图 + 世界地图 `P2` · 2 次迭代

- `sceneWorld / sceneRaid` 双场景机制推广为**场景注册表**；换世界种子即换地图（plan-V2 R2 的 `heightAt` 已经参数化好）。
- 第二区域等级 6-13，含据点（商人 + 墓地 + 旅店）、独立音乐、进入时的**区域名淡入文字**。
- **小地图**：右上角圆形，程序化俯视图（地形顶点色降采样成一张 canvas）+ 光点（自己 / NPC / 任务目标 / 采集物）。
- **世界地图**（`M` 键）：全区域 canvas 绘制 + 任务目标区域高亮。

## 八 · 收尾轨道

### C12 · 副本流程化 + Boss 工厂 + AI 队友 `P1` · 2-3 次迭代

- 把 `combat.js` 的 `bossAI` / 投射物 / 烈焰之子迁入 `js/raid.js`，Boss 抽象为 `createBoss(config)`：技能是数据 `{name, cd, cast, range, effect, vfx}`，阶段是数据 `{hpPct, onEnter}`。
- 副本分段：走廊（成对精英小怪）→ 小 Boss → 岩桥开启 → 终 Boss。这是 WoC 五个副本共用的结构。
- **AI 队友（替代联机的关键一步）**：招募 1-2 名 AI 同伴（复用 C3 的属性块与职业技能）：跟随 / 集火你的目标 / 血低时治疗 / 中断施法。WoC 的五机器人能自动通关副本，说明这条路走得通；对单机项目而言，AI 队友是「五人本内容」唯一可行的解法。
- 小队框架（`ui/frames.js` 复用）+ 队友经验共享。

### C14 · 扩充职业（走向九职业） `P2` · 开放式

C3 的属性系统 + C5 的三套资源 + C7 的法术等级表落地后，**加职业变成纯配置**：一条 `CLASSES` 数据 + 一条法术表 + 一条外观配置（plan-V2 的 `CLASS_LOOK`）。

优先顺序建议：**牧师**（补齐治疗，让 AI 队友和副本成立）→ **盗贼**（验证能量+连击点这套完全不同的资源）→ **猎人**（远程死区 + 宠物）→ 其余。

### C15 · 存档 `P1` · 1 次迭代

- `saveGame() / loadGame()`：只存纯数据（等级、经验、属性、背包、装备、任务表、天赋、金币、坐标、区域），**绝不序列化 Three.js 对象**，加载后按数据重建。
- 主存 localStorage，附带「导出存档码」（JSON → Base64）。
- **格式建议直接按 WoC 的角色 JSONB 结构组织** —— 即便不做联机，一个规整的存档 schema 也让版本迁移与作弊台调试容易得多。

## 十一 · 风险与取舍

| 风险 | 影响 | 缓解 |
| --- | --- | --- |
| **C4 命中表改变全部数值量级** | 现有 `BALANCE` 全部失衡，Boss 打不动或秒杀 | 按「期望 DPS 不变」反推系数；作弊台加 `cheat.dps()` 打桩测试；分开提交（先接表、后调数） |
| **C1 改朝向逻辑影响技能判定** | `heroicStrike` 的 `reach`、`charge` 的贴近可能失效 | 输入抽象为意图对象，技能只读 `S.p.face`；改完跑完整回归 |
| **初版公式常量记错** | 数值「差不多但不对」，比明显错误更难查 | 常量全部进数据表并逐条标注来源与置信度；不确定的先标 `TODO: verify`，别混在函数体里 |
| **系统堆叠导致 `combat.js` 爆炸** | 单文件 1500 行以上不可维护 | 每引入一个大系统就同步拆文件（`sim/` 与 `ui/` 的分层就是为此） |
| **范围失控** | 九职业 + 三区域 + 五副本是几年的工程量 | C14 明确标为开放式；**1-10 级 + 一个区域 + 一个副本 + 三职业**就是一个完整可发布的产品，后面全是加法 |

**明确的取舍**

- **不做联机**。但 `sim/` 分层、确定性随机、纯数据存档三件事都按「将来能联机」的标准做 —— 成本几乎为零，可能性保留。
- **不做职业专精天赋树的完整 51 点版本**。每职业 3 层小树（升级得点、被动修饰器为主）就够，导出 build 字符串留到有人真的需要时。
- **不做宠物系统 / 专业技能 / 声望 / 坐骑**。这些是初版的「厚度」，不是它的「骨架」，等骨架立住再说。
- **不做真实的 5 人本**。用 AI 队友替代 —— 单机项目里这是唯一能让副本内容成立的方案。
